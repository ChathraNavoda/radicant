# Design Tokens

## Purpose

Design tokens create a shared visual language between designers and engineers.

They define meaning instead of implementation.

Widgets should reference tokens rather than hardcoded values.

---

# Color Tokens

| Token | Purpose |
|--------|---------|
| Primary | Brand identity |
| Primary Container | Primary backgrounds |
| Secondary | Supporting UI |
| Accent | Achievements |
| Background | App background |
| Surface | Cards & dialogs |
| Error | Error states |
| Warning | Warning states |
| Success | Positive feedback |
| Info | Informational states |
| Text Primary | Main text |
| Text Secondary | Supporting text |
| Divider | Borders & separators |

# Spacing Tokens

| Token | Value |
|--------|------:|
| xs | 4 |
| sm | 8 |
| md | 16 |
| lg | 24 |
| xl | 32 |
| xxl | 48 |

---

# Motion Principles

Motion should communicate:

- continuity
- hierarchy
- growth

Never use animation purely for decoration.

---

# Icon Principles

- Consistent stroke width.
- Rounded corners where appropriate.
- Simple geometry.
- Easily recognizable at small sizes.
