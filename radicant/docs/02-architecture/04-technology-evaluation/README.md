# Technology Evaluation

## Goal

Select technologies that align with Radicant's long-term vision while remaining sustainable for a solo founder during the MVP stage.

---

# Evaluation Criteria

- Developer Experience
- Cost
- Scalability
- Maintainability
- Community
- Documentation
- Learning Value
- Vendor Lock-in

---

# Frontend

## Flutter

### Decision

✅ Adopt

### Why

- Cross-platform (Mobile + Web)
- Strong existing expertise
- Excellent developer experience
- Single codebase
- Mature ecosystem

---

# Backend

## NestJS

### Decision

✅ Adopt

### Why

- TypeScript
- Modular architecture
- Dependency Injection
- Validation
- OpenAPI support
- Enterprise-ready structure

---

# Database

## PostgreSQL

### Decision

✅ Adopt

### Why

- Relational domain
- Excellent performance
- Strong ecosystem
- Open source
- Portable across providers

---

# ORM

## Prisma

### Decision

✅ Adopt

### Why

- Type safety
- Schema migrations
- Great developer experience

---

# API

## REST

### Decision

✅ Adopt

### Why

- Simplicity
- Excellent Flutter support
- Easy debugging
- Well suited for MVP

### Future Consideration

GraphQL may be introduced if product complexity and client requirements justify it.
