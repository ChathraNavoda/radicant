# Persona 01 — The Growing Software Engineer

## Overview

The Growing Software Engineer is a developer who actively learns through building real software but struggles to preserve the knowledge gained along the journey.

They work across multiple projects, technologies, and teams, continuously solving problems and making technical decisions. While their code is preserved in version control, the reasoning, lessons, and experiences behind that work gradually disappear.

They want a better way to capture and reflect on their engineering growth without interrupting their workflow.

## Goals

- Remember important engineering decisions.
- Capture lessons while building.
- Build a meaningful engineering portfolio.
- Prepare confidently for interviews.
- Demonstrate professional growth over time.
- Build reusable engineering knowledge.
- Become a better software engineer every year.

---

## Frustrations

- Important ideas are forgotten.
- Notes become scattered across multiple tools.
- Git history explains what changed but rarely why.
- Preparing a CV or portfolio requires reconstructing months of work.
- Valuable engineering lessons disappear after projects end.

---

## Behaviors

- Uses GitHub daily.
- Uses AI as a learning partner.
- Frequently researches documentation.
- Saves useful resources but rarely revisits them.
- Reflects mentally but rarely records those reflections.

---

## Needs

The user needs a system that naturally fits into the software development workflow.

The system should require minimal effort while continuously building an engineering memory.

Reflection should feel lightweight rather than like maintaining documentation.

---

## Success Looks Like

Six months after using Radicant, the user can:

- Recall why important technical decisions were made.
- Reuse previous solutions instead of solving the same problems again.
- Generate portfolio content from real engineering experiences.
- Prepare for interviews using authentic project stories.
- Clearly see how they have grown as an engineer.
