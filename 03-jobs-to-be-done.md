# Jobs To Be Done

## Core Job

When I solve problems and learn while building software,

I want to effortlessly capture and organize my engineering knowledge,

So that I can remember how I became the engineer I am today.

---

## Functional Jobs

- Capture engineering insights while working.
- Record architectural decisions.
- Preserve debugging experiences.
- Track project evolution.
- Organize technical knowledge.
- Search previous experiences.
- Build reusable engineering knowledge.

---

## Emotional Jobs

- Feel confident before interviews.
- Feel proud of personal growth.
- Never feel like valuable experience has been wasted.
- Build confidence through reflection.

---

## Social Jobs

- Share engineering stories.
- Showcase growth to recruiters.
- Mentor other developers.
- Demonstrate engineering thinking beyond code.

---

## Success Criteria

A user should naturally think:

"I've already solved something similar before."

And find it within seconds.
